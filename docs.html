Termi Chat — Quick Start ✨

A tiny terminal-style chat with a Matrix background. This page tells you how to use it, test it locally, and deploy it. Friendly, short

What is Termi Chat? 💚

Termi Chat is a static frontend that looks like a hacker terminal. It uses Firebase Firestore to store messages in realtime. You type into the command line, press Enter, and messages appear in the chat like a terminal log.

How to use — quick guide ▶️

  1. Open the site and you'll see a prompt: enter a username (just type and press Enter). 👤
  2. Once logged in, type a message and press Enter to send. Your message will appear in the chat and others will see it in realtime. 💬
  3. Use these local commands (start with /):
    • /nick NEWNAME — change your display name ✏️
    • /say MESSAGE — broadcast a message without changing prompt 🗣️
    • /clear — clear your local chat window 🧹
    • /help — show available commands ❓
  4. If you like the Matrix background, there's a toggle labeled Pause Matrix in the UI to pause/resume it. ⚡

Developer notes 🛠️

Files you care about:

  • index.html — entry page and UI layout 📄
  • script.js — main client logic, Firebase hookup, message rendering 🧩
  • matrix.js — animated Matrix background 🎞️
  • style.css — visuals and responsive behavior 🎨

Data shape & realtime behavior 🔁

Messages are stored in Firestore in a messages collection. Each message looks like:

{
  username: "alice",
  message: "Hello world",
  timestamp: "2025-10-07T12:34:56.789Z"
}

The client listens to firestore updates and appends new messages in realtime. A small optimization ensures locally echoed messages are replaced smoothly when Firestore confirms them (no flicker ✨).

Local testing 🧪

Run a simple static server from the repo root and open the site in your browser:

# Python 3 (recommended)
python3 -m http.server 8000

# Or with Node (http-server)
npx http-server -p 8000

Then open http://localhost:8000 in your browser and try sending messages. 🖥️➡️🌐

Setup Firebase 🔐

  1. Create a Firebase project at console.firebase.google.com. 🧾
  2. Add a Web App in Project Settings and copy the config values (apiKey, projectId, ...). ⚙️
  3. Enable Firestore and create a collection named messages. 📚
  4. Paste your config into the firebaseConfig object in script.js. 🔁
  5. For demos you can allow open reads/writes; for production protect your DB with rules or require auth. 🔒

Deploying ✅

GitHub Pages works great for this static repo:

  1. Push your repo to GitHub.
  2. In repository Settings → Pages, select branch main and folder / (root).
  3. Your site will be live at https://<your-user>.github.io/<repo>/. 🎉

Safety & tips ⚠️

  • Do not commit sensitive API keys to public repos — rotate or move to a secure place. 🔑
  • Use Firestore security rules before using this publicly — otherwise anyone can read/write messages. 🛡️
  • If you see visual flicker when sending messages, try the latest browser or test locally with a simple server (the app replaces local echoes smoothly to avoid flashing). 🧯
