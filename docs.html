Data shape & realtime behavior ğŸ”

Messages are stored in Firestore in a messages collection. Each message looks like:

{
  username: "alice",
  message: "Hello world",
  timestamp: "2025-10-07T12:34:56.789Z"
}

The client listens to firestore updates and appends new messages in realtime. A small optimization ensures locally echoed messages are replaced smoothly when Firestore confirms them (no flicker âœ¨).
