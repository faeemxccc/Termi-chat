Termi Chat — Documentation

Quick guide: how this site is built, where data lives, and how to deploy it.

Why I made this

This project was created for fun and learning. It mimics a hacker-style terminal chat UI while practicing frontend interactivity, realtime databases, and static site hosting. The green-on-black aesthetic is just for the vibe — it's playful and retro.

High-level architecture

  • Frontend: static HTML/CSS/JS (this repo)
  • Realtime backend: Firebase Firestore (used to store messages)
  • Hosting: GitHub Pages serves the static site

Firebase (Firestore) setup

  1. Create a Firebase project at console.firebase.google.com.
  2. In Project Settings, add a Web App and copy the config snippet (apiKey, projectId, etc.).
  3. Enable Firestore in the console and create a collection named messages.
  4. Security: for quick demos you might allow open reads/writes, but for production set rules to require authentication or server-side validation.
  5. In script.js replace the firebaseConfig object with your app's values and keep the firestore compat script included in index.html.

Data shape

{
  username: "alice",
  message: "Hello world",
  timestamp: "2025-10-07T12:34:56.789Z"
}

Deploy to GitHub Pages

  1. Push your repo to GitHub.
  2. In the repo Settings → Pages, choose the branch (main) and folder (root) to publish.
  3. Your site will be available at https://<your-user>.github.io/<repo>/ shortly.

Local testing

Open index.html in your browser or run a simple static server (recommended) to avoid any file-origin limitations for modern APIs:

# Python 3
python -m http.server 8000

Notes & tips

  • Use Firestore security rules for any real deployment.
  • Remove or rotate API keys if you share the repo publicly, or move them to environment-protected systems.
  • GitHub Pages is ideal for static frontends but not for private or server-side logic.
